Software Engineering Intern

I had the opportunity to work with Exegy, a high-frequency trade data normalization firm as a software engineering intern in the Client API (XCAPI) team under Mike Kohn. I worked on several projects during my time at Exegy, and I would discuss them here but in a very general sense so as to comply with the NDA.

API Profiling

The first task I was assigned was to demonstrate the performance differences between the C and JAVA APIs as well as the how the differences between the current master branch and the newer feature branches, which promise improvement. As expected, the feature branches were significanlty better. I also had to validate the data I was testing, for which I wrote a script that compares the hashes of different request bodies for the feature branches and the already tested Master branch. Technical factors I took into consideration while profiling were

  • Data equivalency: Two processes corresponding to different API versions are compared on the closest matching data
  • Process Isolation: Both processes run on separate cores, and separate NUMA strips to avoid caching artifacts
  • Removal of outliers: To remove artifacts due to non-reproducable events

Web interface

At Exegy, we frequently used a command-line based testing utility. Binary editor

The last project I worked on involved me writing a software library which will allow an existing utility to be able to edit binaries containing C/C++ objects. I wrote a script which would take as input the structure definition headers, and parse it to extract useful attributes such as type, identifiers, size etc. The script also needed to handle bit-fields, nested structures, and nested unions, and be able to edit every field inside those sub-structures as well. After parsing it, the script would generate C/C++ style code, which would constitute the software library I was supposed to develop, allowing for scalalbilty if more strucutures are modified, introduced or removed.
