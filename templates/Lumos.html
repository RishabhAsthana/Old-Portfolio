Lumos

Music Visualizer

Built in Processing (2.7), using minim and sDrop

I worked on Lumos during the summer of my sophomore year. Having worked on a visualizer before, I figured out changes that would make the user experience better, more intuitive, and having the knowledge from the TAV, it was easy to get started on this project.

The areas I focused on this time around were centered around to make the visualizer a 'self-sufficient' entity, such that it wouldn't need any user input to function, however the user can request certain changes if they feel like it, and the visualzer would accomodate.

This visualizer can run on 1080p at 60fps on a 3rd gen i5 3337u, however it still suffers with the problem of portability, which I think I would be able to solve in another iteration, using WebGL
